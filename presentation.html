üéØRunnable Interface

The Runnable interface should be implemented by any class whose instances are intended to be executed by a thread.

  • ‚úÖ Functional interface with a single method run()
  • ‚úÖ Preferred way to create threads (more flexible)
  • ‚úÖ Allows class to extend another class while implementing Runnable
  • ‚úÖ Better object-oriented design (implements vs extends)
@FunctionalInterface
public interface Runnable {
// Must be implemented by classes that want to run in a thread
public abstract void run();
}

üßµMyRunnable Class Implementation

class MyRunnable implements Runnable {
@Override
public void run() {
try {
// Print thread information
System.out.println("Thread " + Thread.currentThread().getId() + " is running.");

// Suspend thread for 500 milliseconds
Thread.sleep(500);
}
catch (InterruptedException e) {
System.out.println("Thread interrupted: " + e.getMessage());
}
}
}
Thread.sleep(500) suspends the thread for 500 milliseconds
Thread.currentThread().getId() returns the unique thread ID

üöÄThreadMain Class Implementation

public class ThreadMain {
public static void main(String[] args) {

// Create and start 5 threads

}
}
}
new Thread(new MyRunnable()) creates a new thread
thread.start() starts the thread and calls the run() method

‚öñÔ∏èRunnable vs Thread Extension

Two ways to create threads in Java:

// Method 1: Implementing Runnable (Preferred)
class MyRunnable implements Runnable {
public void run() { ... }
}
Thread t = new Thread(new MyRunnable());

// Method 2: Extending Thread (Less flexible)
class MyThread extends Thread {
public void run() { ... }
}
MyThread t = new MyThread();

Why Runnable is preferred: Allows class to extend another class, better OOP design, more flexible!
